## Validations

The validation logic hasn't been integrated in Field on purpose so consumers can provide their own custom validations.

Field accepts an array of validation messages that will be displayed in an attached InlineAlert component. A set of default translated messages are provided and can be access via id. See list of supported error messages [here](https://github.com/transferwise/neptune-web/blob/main/packages/components/src/Field/Fiels.messages.ts)

Validation messages are automatically attached to the children based on user interaction and with styling.
